# ProjectHub - Firestore Database Structure

This document outlines the NoSQL database schema used in the ProjectHub application, powered by Firebase Firestore.

## Collections & Documents

### 1. `users`
Stores user profile information and role definitions.
*   **Document ID**: User's Firebase Auth UID
*   **Fields**:
    *   `email` *(string)*: User's email address (used as a primary identifier across the app)
    *   `name` *(string)*: Full name
    *   `role` *(string)*: One of `"student"`, `"faculty"`, `"hod"`, or `"admin"`
    *   `department` / `dept` *(string)*: Associated department (e.g., "CSE", "IT")
    *   `studentId` *(string, optional)*: Roll number or student ID (for students only)
    *   `createdAt` *(timestamp/string)*: Account creation date

### 2. `departments`
Stores department-level metadata, primarily used by Admins for auto-generating student IDs.
*   **Document ID**: Department Code (e.g., "CSE")
*   **Fields**:
    *   `currentStudentCount` *(number)*: Counter used to generate sequential student roll numbers.

### 3. `projects`
The core entity representing an academic project.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `name` *(string)*: Project title
    *   `description` *(string)*: Brief overview of the project
    *   `tech` *(string)*: Technology stack (e.g., "React, Node.js")
    *   `outcome` *(string)*: Expected outcome of the project
    *   `facultyId` *(string)*: Email of the assigned Faculty Guide
    *   `facultyName` *(string)*: Name of the assigned Faculty Guide
    *   `team` *(array of strings)*: List of student emails participating in the project
    *   `dept` *(string)*: Department the project belongs to
    *   `status` *(string)*: Current state (e.g., `"Pending"`, `"In Progress"`, `"Completed"`, `"Rejected"`)
    *   `progress` *(number)*: Completion percentage (0-100)
    *   `milestone` *(string)*: Current active milestone (e.g., `"Awaiting Approval"`)
    *   `color` *(string)*: Hex code for UI representation (e.g., `"#4f46e5"`)
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 4. `tasks`
Kanban board tasks associated with a specific project.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the parent project
    *   `title` *(string)*: Task description
    *   `status` *(string)*: Current column (`"Todo"`, `"In Progress"`, `"Review"`, `"Completed"`)
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 5. `feedback`
Communication threads and feedback between faculty and students on a project.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the parent project
    *   `text` *(string)*: The feedback message content
    *   `status` *(string)*: State of the feedback (`"Open"`, `"Resolved"`)
    *   `author` *(string)*: Name of the user who posted the feedback
    *   `role` *(string)*: Role of the author (`"student"`, `"faculty"`)
    *   `readBy` *(array of strings)*: List of emails of users who have viewed this message
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 6. `notifications`
Alerts and updates targeted at specific users.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `userId` *(string)*: Email of the recipient user
    *   `title` *(string)*: Notification header
    *   `message` *(string)*: Detailed notification content
    *   `type` *(string)*: Category of notification (e.g., `"proposal"`, `"feedback"`)
    *   `projectId` *(string, optional)*: Reference to the relevant project, if applicable
    *   `read` *(boolean)*: Whether the user has acknowledged the notification
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 7. `files`
Metadata for files and documents uploaded for a project (actual files are stored in Firebase Storage or Cloudinary).
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the parent project
    *   `name` *(string)*: Filename or demo link description
    *   `url` *(string)*: URL to the hosted file or external link
    *   `uploadedBy` *(string)*: Name of the uploader
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 8. `updates`
Progress updates and milestones posted by the team.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the parent project
    *   `content` *(string)*: Description of the update
    *   `author` *(string)*: Name of the user posting the update
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 9. `activityLogs`
Audit trail of actions taken within a project workspace.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the parent project
    *   `action` *(string)*: Description of the action (e.g., "File Uploaded")
    *   `user` *(string)*: Name/email of the user who performed the action
    *   `createdAt` *(timestamp)*: Firestore server timestamp

### 10. `invitations`
Pending team member invitations.
*   **Document ID**: Auto-generated by Firestore
*   **Fields**:
    *   `projectId` *(string)*: Reference to the project
    *   `email` *(string)*: Email of the invitee
    *   `status` *(string)*: Current state (`"pending"`, `"accepted"`, `"rejected"`)
    *   `createdAt` *(timestamp)*: Firestore server timestamp
